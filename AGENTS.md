# AGENTS.md — AI Agent Coding Conventions

This file defines rules and conventions for AI coding agents (Cursor, Copilot, Claude, etc.) working in the AprilAlgo repository.

---

## Project Identity

- **Name:** AprilAlgo
- **Type:** Python data science / analysis project
- **Package manager:** `uv`
- **Python:** 3.11+
- **License:** Apache 2.0

---

## Non-Negotiable Rules

1. **Never edit `uv.lock` manually.** It is auto-generated by `uv`. Run `uv add` / `uv remove` / `uv sync` instead.
2. **Never use `pip install`.** All package management goes through `uv`.
3. **Never commit `.venv/`.** It is in `.gitignore` and is always recreated via `uv sync`.
4. **Always use `uv run` to execute scripts**, not `python` directly, to ensure the virtual environment is active.
5. **Do not edit `.python-version` unless intentionally upgrading the Python version.**

---

## Adding Dependencies

```bash
# Add a runtime dependency
uv add <package>

# Add a development-only dependency
uv add --dev <package>

# Remove a dependency
uv remove <package>
```

After adding/removing, `pyproject.toml` and `uv.lock` will be updated automatically.

---

## File Structure Conventions

| Location                      | Purpose                                     |
|-------------------------------|---------------------------------------------|
| `src/aprilalgo/`              | Importable Python package code              |
| `main.py`                     | Top-level entry point script                |
| `notebooks/`                  | Jupyter notebooks (create when needed)      |
| `data/raw/`                   | Original, unmodified data files             |
| `data/processed/`             | Cleaned / transformed data                  |
| `tests/`                      | Unit tests (create when needed)             |

---

## Code Style

- Follow **PEP 8**.
- Maximum line length: **100 characters**.
- Use **type hints** for all function signatures.
- Use **docstrings** for all public functions and classes (Google style preferred).
- Prefer **f-strings** over `.format()` or `%` formatting.
- Use `pathlib.Path` for file paths, not raw string concatenation.

---

## Changelog

When making changes, add an entry to `CHANGELOG.md` under `[Unreleased]` using this format:

```markdown
### Added
- Description of new feature

### Changed
- Description of modification

### Fixed
- Description of bug fix

### Removed
- Description of removed feature
```

---

## Version Bumping

Version is defined in `src/aprilalgo/__init__.py` as `__version__` and in `pyproject.toml` as `version`. Both must be updated together when releasing.

---

## Running the Project

```bash
# Install all dependencies (first time or after pulling changes)
uv sync

# Start Jupyter
uv run jupyter notebook

# Run the main script
uv run python main.py
```
