## What We Installed on Your Computer

Before creating the project, we had to install two tools on your Windows machine.

### 1. Rust
Rust is a programming language вЂ” but you don't need to write any Rust. We installed it because some tools (like `uv`) are built with Rust and need it to work. Think of it like needing a specific engine under the hood of a car even if you never look at it.

- Installed with: `winget install Rustlang.Rustup`
- Version installed: 1.29.0

### 2. uv
`uv` is your new best friend for Python projects. It does several jobs at once:
- Downloads and installs Python packages (like `pandas`, `numpy`)
- Creates a "virtual environment" (an isolated workspace just for your project)
- Keeps track of exactly which version of every tool you're using

Old way (what most tutorials show): `pip install ...`
New way (what professionals use): `uv add ...`

- Installed with: `winget install --id=astral-sh.uv -e`
- Version installed: 0.11.2

### README.md
The "front door" of your project. If you ever share this with someone on GitHub, this is the first thing they read. It explains what the project does and how to use it. You should keep it updated as your project grows.

### LICENSE
A legal document that says "Apache 2.0" вЂ” this is a popular open-source license. It means: anyone can use your code, copy it, or build on it, as long as they give you credit. Your name (Joshua) and the year (2026) are included.

### .gitignore
A hidden file (the dot at the start means it's hidden). It tells Git вЂ” the tool used to save and track your code history вЂ” which files to **ignore**. For example, your `.venv` folder has thousands of files but you never want to upload those to GitHub. This file handles that automatically.

### CHANGELOG.md
A diary for your project. Every time you make a meaningful change, you add an entry here. We already added the first entry: `v0.1.0 вЂ” 2026-03-29` (your initial setup). The format follows an industry standard called "Keep a Changelog."

### CLAUDE.md
Special instructions for **Claude AI** (the AI assistant behind tools like this one). When you open this project in Cursor and ask Claude for help, it reads this file to understand your project вЂ” what it does, how commands work, where things are. Think of it as a briefing document for your AI assistant.

### AGENTS.md
Similar to CLAUDE.md, but written for **any** AI coding agent (like Cursor's built-in AI). It lists the rules: never use `pip`, always use `uv run`, where to put files, how to write code. It keeps the AI behaving consistently.

### pyproject.toml
The "ID card" of your Python project. It records:
- The name of your project (`aprilalgo`)
- The version (`0.1.0`)
- What Python version is needed (`>=3.11`)
- What packages your project depends on (`pandas`, `numpy`, etc.)

The `.toml` format is just a simple text format вЂ” like a configuration file.

### uv.lock
This was automatically created by `uv`. It records the **exact version** of every single package installed (106 packages total). This is important: if someone else downloads your project and runs `uv sync`, they get the exact same versions you have. Think of it like a receipt that lists every item you purchased, with specific serial numbers.

Do not edit this file manually. `uv` manages it.

### .python-version
A tiny file that just says which version of Python this project uses. `uv` reads it automatically.

### .venv/
Your **virtual environment** вЂ” a self-contained folder with your Python installation and all your packages. It's completely isolated from the rest of your computer. If something breaks here, it doesn't affect anything else.

This folder is **not tracked by Git** (the .gitignore excludes it). Anyone who clones your project recreates it by running `uv sync`.

### main.py
The entry point of your program вЂ” the file you run to start things. Right now it has a basic "Hello, World!" style starting point. This is where you'll write your data analysis code, or import from the `src/aprilalgo/` package.

### src/aprilalgo/__init__.py
This file makes `aprilalgo` a proper Python "package" вЂ” meaning other Python files can import code from it using `import aprilalgo`. It contains your project's version number. As your project grows, you'll add more `.py` files inside `src/aprilalgo/`.

## How to Use Your Project Day-to-Day

All commands are run inside the `AprilAlgo` folder in a terminal (PowerShell).

### Open a Jupyter Notebook (recommended for data work)
```powershell
cd C:\Users\joshu\coding\AprilAlgo
uv run jupyter notebook
```
This opens a browser tab where you can write and run Python code interactively, see charts inline, and explore data visually.

### Run the main script
```powershell
uv run python main.py
```

### Install a new package
```powershell
uv add seaborn
```
Replace `seaborn` with whatever package you need. `uv` will install it, update `pyproject.toml`, and update `uv.lock` automatically.

### Restore everything after downloading on a new machine
```powershell
uv sync
```
This reads `uv.lock` and installs all packages exactly as they were.

## Glossary (Plain English)

| Term | What it means |
|------|--------------|
| **Package** | A bundle of code someone else wrote that you can use in your project |
| **Virtual environment (.venv)** | An isolated bubble of Python just for this project |
| **Git** | A tool that saves snapshots of your code over time (like unlimited undo) |
| **GitHub** | A website where you upload your Git code history to share or back it up |
| **Terminal / PowerShell** | The black/blue window where you type commands |
| **Dependency** | A package your project needs to work |
| **Version (0.1.0)** | A label for "how finished" your project is. 0.x = early stage. 1.0 = stable release |
| **Import** | In Python, "import pandas" means "bring the pandas package into this file so I can use it" |
